## Core Concepts

Before we learn how to process files, we need to understand what data looks like inside Python and how it is structured within external files.

### Python Dictionaries (`dict`)

Think of a real-world dictionary: you look up a "word" to find its "definition." In Python, a **Dictionary (or `dict` for short)** works exactly the same way. It is a container used to store data in **Key-Value pairs**.

* **Key**: Like the word in a dictionary, it acts as a label. Keys are usually text strings (e.g., "name", "price").
* **Value**: Like the definition of the word, this is the actual data you want to store. It can be a number, a text string, a boolean (True/False), or even another list or dictionary.

**Example:**
```python
# This is a Python dictionary containing product information
product = {
    "name": "Laptop",    # The Key is "name", the Value is the string "Laptop"
    "price": 1200,       # The Key is "price", the Value is the number 1200
    "in_stock": True     # The Key is "in_stock", the Value is the boolean True
}

# We can retrieve the Value by using its Key
print(product["name"])   # Output: Laptop
```
Understanding Python dictionaries is crucial because the CSV and JSON data we are about to introduce are typically converted into dictionaries once loaded into Python!

### File Handling Basics

1. The `open()` Function
   `open()` is the built-in tool Python uses to open a connection to a file. It uses three primary parameters:
   * **File Path**: The location and name of the file.
   * **Mode**: Tells Python what you intend to do. Common modes include:
     * `'r'` (Read): To read the content (default).
     * `'w'` (Write): To write content (overwrites existing data).
     * `'a'` (Append): To add content to the end of the file.
   * **Encoding**: It is highly recommended to set `encoding='utf-8'` to ensure that special characters and different languages are handled correctly.

2. The `with` Statement (Context Manager)
   While you can open a file using `file = open(...)`, you must remember to manually call `file.close()` afterward to release computer resources. If an error occurs in your code before that line, the file might remain open, leading to potential data corruption or memory leaks.

   **Why use `with`?**
   Python recommends the `with` keyword, known as a **Context Manager**. It provides two major benefits:
   * **Automatic Closing**: Python will **automatically** close the file for you as soon as the code inside the `with` block finishes, even if an error occurs.
   * **Cleaner Code**: You don't need to write `close()`, making your scripts more concise and safer.
   
   **Basic Syntax Example:**
   ```python
   # Open a file using 'with' and nickname it 'f'
   with open('example.txt', mode='r', encoding='utf-8') as f:
       content = f.read()
       print(content)
   
   # The file is automatically closed once we leave this indentation block
   ```

### CSV (Comma-Separated Values)

**What is it?**
You can think of a CSV file as a "plain-text Excel spreadsheet." It is one of the oldest and most universal formats for storing tabular data. If you strip away all the fancy borders, fonts, and colors from a spreadsheet, leaving only the raw data, you get a CSV:
1. **Newlines** represent a "Row" (a single data record).
2. **Commas** are used to separate the different "Columns" (the fields).

**Visual Comparison:**
If your spreadsheet looks like this:

| id | name     | price |
|----|----------|-------|
| 1  | Laptop   | 1200  |
| 2  | Mouse    | 25    |
| 3  | Keyboard | 75    |

Its plain-text appearance in a CSV file will look exactly like this:
```csv
id,name,price
1,Laptop,1200
2,Mouse,25
3,Keyboard,75
```

**Important Formatting Mechanisms (Dialect Parameters)**
Despite the name "Comma-Separated," real-world data files generated by different software often have slight formatting variations. Before looking at how Python reads these files, we need to understand three core formatting mechanics. Python's `csv` module allows us to customize these rules to ensure data is parsed and written accurately:

* **Delimiter (`delimiter`)**: The character used to separate fields. It defaults to a comma (`,`), but if your specific file requires it, you can easily change this to a semicolon (`;`) or a tab (`\t`).
* **Quote Character (`quotechar`)**: If your actual data contains the delimiter itself (for example, an address like "New York, NY" containing a comma), that specific field needs to be wrapped in quotes. This prevents the program from mistakenly splitting the address into two separate columns. It defaults to double quotes (`"`), but can be changed to single quotes or other symbols as needed.
* **Line Terminator (`lineterminator`)**: This determines the hidden character used to signal the end of a row when writing files. The default usually depends on your operating system (e.g., Windows typically uses `\r\n`). However, explicitly defining it when writing a file (such as `lineterminator='\n'`) can prevent unexpected blank lines when sharing files across different platforms.

**How does Python handle it?**
Python comes with a built-in `csv` module to help us easily read and write these files. Depending on how you want to process the data, there are two main approaches:

**1. Basic Processing: Handling Data as Lists**
This approach is perfect for handling files without headers or files with very simple structures.
* **`csv.reader`**: When reading a CSV file, it converts each row into a Python List containing strings. You can retrieve specific data by using indexes (e.g., `row[0]`, `row[1]`).
* **`csv.writer`**: Takes Python Lists and writes them directly into the CSV format. You can use `writerow()` to write a single row or `writerows()` to write multiple rows at once.

**2. Advanced Processing: Handling Data as Dictionaries**
If your CSV file has a header row, using this method is highly recommended. It makes your code much more intuitive and readable.
* **`csv.DictReader`**: Smartly uses the first row (the headers) as Dictionary Keys, and converts every subsequent row into a Python Dictionary. This means you can access data using column names (like `row['price']`) instead of memorizing index positions.
* **`csv.DictWriter`**: Takes your Python Dictionary data and writes it back into the correct CSV format, automatically matching it to the corresponding header row.

**Code Example: Basic Reading and Writing with Custom Symbols**
```python
import csv

# Preparing list data to write (Notice: we are using Lists here, not Dictionaries)
data = [
    ["id", "name", "location"],
    [1, "Alice", "New York, NY"],  # The comma here requires a quotechar to protect it
    [2, "Bob", "London"]
]

# 1. Writing to a CSV file (setting custom dialect parameters)
# Note: Adding newline='' is recommended by the official Python documentation 
# so the csv module can safely manage the line terminations on its own.
with open('custom_data.csv', mode='w', newline='', encoding='utf-8') as file:
    writer = csv.writer(file, delimiter=',', quotechar='"')
    writer.writerows(data)

# 2. Reading from a CSV file
with open('custom_data.csv', mode='r', newline='', encoding='utf-8') as file:
    reader = csv.reader(file, delimiter=',', quotechar='"')
    for row in reader:
        # 'row' will be a Python list, e.g., ['1', 'Alice', 'New York, NY']
        print(f"ID: {row[0]}, Name: {row[1]}, Location: {row[2]}")
```

### JSON (JavaScript Object Notation)

**What is it?**
Even though it has "JavaScript" in its name, JSON is now the "universal language" of the internet. When you scroll through social media or check a weather app, the data sent from the servers is almost always in JSON format.

JSON was designed to be easily parsed by machines, while still being easily readable by humans. **The best part for Python beginners is that JSON looks almost exactly like a Python Dictionary!**

**Visual Comparison:**
A piece of JSON data inside a file looks like this (wrapped in curly braces):
```json
{
    "id": 1,
    "name": "Laptop",
    "price": 1200
}
```

**How does Python handle it?**
Python has a built-in `json` module that allows you to seamlessly transition between "Python Dictionaries" and "JSON files."
* `json.load()`: Reads data from a JSON **file** and automatically converts it into a Python dictionary.
* `json.dump()`: Takes your Python dictionary and exports it, saving it as a standard JSON **file**.
* `json.loads()` / `json.dumps()`: The extra `s` stands for "string." You use these two tools when you want to convert between a Python dictionary and raw JSON text directly in your code, rather than interacting with a file.

## Guided Practice

Imagine we have already obtained an inventory list from the company's system, named **`inventories.csv`**. This file contains product details including `id`, `name`, `price`, and `quantity`. 

The front-end engineers need this data to build a website, but they specifically require it to be in **JSON format**. Our mission is to write a Python script that reads this CSV file, ensures the numbers are the correct data type, and exports everything into a clean JSON file. 

Let's walk through the logical steps required to build this script.

### Step 1: Setting up Tools and Storage
Before we interact with any files, we need to prepare our environment. 
First, we must import Python's built-in modules designed specifically for handling CSV and JSON formats. 
Next, we need to create an empty Python List. 
Think of this List as a temporary storage box where we will place our data once we extract it from the CSV file.

### Step 2: Reading the CSV File
We will use the context manager (the `with` statement) to open `inventories.csv` in read mode. 
To make data manipulation intuitive, we should use the specific CSV tool that reads the file and automatically converts each row into a Python Dictionary, using the top row (headers) as the dictionary keys. 

**Crucial Concept:** When Python reads a CSV, it views *everything* as plain text (Strings). This means a price like `15.99` is read as the text `"15.99"`, and a quantity of `50` is read as the text `"50"`. We cannot do math with text, so we need to fix this in the next step.

### Step 3: Data Transformation (Type Casting)
We will create a `for` loop to go through the file, row by row. Inside this loop, we need to clean our data:
1. Target the `price` value and convert it from a string into a floating-point number (`float`).
2. Target the `quantity` value and convert it from a string into an integer (`int`).

Once a row has been properly converted, we take that cleaned dictionary and append (add) it to the empty storage List we created back in Step 1.

### Step 4: Exporting to JSON
Once the loop finishes, our List is completely full of clean, correctly typed dictionaries. The final step is to save this out. We use the context manager again to open a brand-new file called `inventory.json` in write mode. 

Using the JSON module, we take our entire List and dump it into this new file. To make the final file easy for humans to read, we can apply an indentation setting (usually 4 spaces) so the JSON data is neatly formatted rather than squashed onto a single line.
